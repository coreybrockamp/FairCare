# Dispute Letter Generator - Implementation Summary

## Overview
Built a complete dispute letter generator feature for FairCare that creates professional medical bill dispute letters using Claude AI and integrates with the bill review workflow.

## Components Created

### 1. **Supabase Edge Function** (`supabase/functions/generate-dispute-letter/index.ts`)
- **Endpoint**: POST to `generate-dispute-letter`
- **Inputs**: 
  - `billData`: Parsed bill details (total due, subtotal, patient responsibility, line items)
  - `errors`: Array of detected billing errors with type, severity, description, estimated overcharge
  - `patientName`: Patient name for letter personalization
  - `providerName`: Provider/facility name
- **Logic**:
  - Builds error summary with CPT codes and overcharge amounts
  - Calls Claude 3 Haiku API with professional system prompt
  - System prompt instructs letter to be firm but professional, reference specific codes/amounts, cite patient billing rights
  - Returns plain text dispute letter
- **Features**:
  - CORS headers configured
  - Error handling with detailed responses
  - Validates required fields before processing

### 2. **Service Layer** (`src/services/disputeGenerator.ts`)
- **Function**: `generateDisputeLetter(billId, errors, patientName, providerName): Promise<string>`
- **Flow**:
  1. Fetches bill from Supabase using billId
  2. Calls edge function via `supabase.functions.invoke()`
  3. Returns generated letter as string
  4. Saves letter to `disputes` table with status='draft'
  5. Returns letter content even if table save fails
- **Error Handling**: Comprehensive try-catch with console logging

### 3. **DisputePreview Screen** (`src/screens/DisputePreview.tsx`)
- **Purpose**: Display generated dispute letter with full functionality
- **Layout**:
  - Header with back button and "Your Dispute Letter" title
  - Scrollable letter content in white card with soft shadow
  - Two action buttons at bottom: Copy Letter, Share
- **Features**:
  - **Copy Button**: Copies letter to clipboard, shows "✓ Copied" feedback
  - **Share Button**: Uses React Native Share API for native sharing (email, messaging, etc.)
  - **Design**: Follows warm FairCare aesthetic (coral #FF6B6B, cream background, white cards)
  - **Mobile-first**: Touch targets properly sized (hitSlop on back button)
  - **Typography**: 14px body text, 22px line height for readability

### 4. **Updated BillResults Screen** (`src/screens/scan/BillResults.tsx`)
- **New Button**: "Generate Dispute Letter" button below "Add EOB"
- **Behavior**:
  - Shows loading indicator while generating
  - Disabled during generation
  - Calls `generateDisputeLetter()` with bill ID, errors array, patient name, provider name
  - Navigates to DisputePreview on success
  - Shows error alert on failure
- **Styling**: Matches existing coral button style

### 5. **Navigation Update** (`src/navigation/ScanNavigator.tsx`)
- Added `DisputePreview` to `ScanStackParamList` type with `{ letter: string }` params
- Registered DisputePreview screen in Stack Navigator

### 6. **Database Migration** (`supabase/migrations/20260311_create_disputes_table.sql`)
- **Table**: `public.disputes`
- **Columns**:
  - `id`: UUID primary key
  - `user_id`: References auth.users (cascade delete)
  - `bill_id`: References bills (cascade delete)
  - `letter_content`: Text containing the dispute letter
  - `status`: Enum field (draft/submitted/resolved/rejected, default 'draft')
  - `created_at`: Timestamp (default now)
  - `updated_at`: Timestamp (default now)
- **Indexes**: On user_id and bill_id for fast queries
- **RLS Policies**: Users can only CRUD their own disputes

## User Flow

1. **User views bill in BillResults screen**
2. **Error detection shows detected errors** (if any)
3. **User taps "Generate Dispute Letter" button**
4. **Loading state shows while Claude generates letter**
5. **DisputePreview screen displays professional letter**
6. **User can**:
   - Read full letter in scrollable view
   - Copy to clipboard for sending manually
   - Share via native Share sheet (email, iMessage, etc.)
7. **Letter is saved to disputes table** in draft status for future reference

## Design Decisions

### Letter Format
- Professional business letter format generated by Claude
- Includes date, patient info, provider info
- References specific CPT codes and dollar amounts from detected errors
- Cites relevant billing rights and regulations
- Sets 30-day response deadline
- Actionable and firm tone

### UI/UX
- Letter in prominent white card on cream background for readability
- Copy button provides quick clipboard access for pasting into emails
- Share button allows native iOS sharing workflows
- Loading state prevents user confusion during generation
- Error handling with user-friendly alerts

### Data Persistence
- Disputes saved with draft status for user review before sending
- Future enhancement: Add "Submit" button to change status to "submitted" and track provider responses

## Technical Stack
- **Backend**: Supabase Edge Functions (Deno)
- **AI**: Claude 3 Haiku via Anthropic API
- **Database**: PostgreSQL (Supabase)
- **Frontend**: React Native / Expo
- **Navigation**: React Navigation stack
